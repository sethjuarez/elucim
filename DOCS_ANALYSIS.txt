## 6. WRITING STYLE & CONVENTIONS

### Structure Patterns:

1. **Guide Pages:**
   - Intro paragraph explaining purpose
   - Hierarchical headings (H1, H2, H3)
   - Code blocks in TSX/TS notation
   - Inline code for props/types
   - Prop tables (| Prop | Type | Description |)
   - Examples with real output
   - Cross-linking to related pages with [text](/path)

2. **API Pages:**
   - Interface definitions with interface keyword
   - Inline  @default  comments
   - Function signatures with  async  notation
   - Type definitions
   - Minimal descriptive text

3. **Code Examples:**
   - Use TSX/TS syntax (not plain JS)
   - Import statements shown
   - Real-world scenarios with meaningful variable names
   - Comments in code for clarity
   - Frame-based durations (not seconds in most examples)

### Frontmatter:

- Minimal - only layout declarations
- Home page uses layout: home with hero config
- Regular pages have implicit default layout

### Markdown Features Used:

- Code blocks with language hints (tsx, ts, html, bash)
- Tables for API reference (pipes |)
- Links using [text](/path)
- Blockquotes for tips using ::: tip
- Emphasis with **bold** and *italic*
- Lists (ordered and unordered)
- Headings up to H3 (sometimes H4 for subsections)

### Color Palette in Examples:

- Primary: #4ecdc4 (cyan/teal)
- Secondary: #ff6b6b (coral red)
- Tertiary: #ffe66d (yellow)
- Quaternary: #a29bfe (purple)
- Text: #e0e0e0, #fff (light colors)
- Dark: #111, #111127, #000 (backgrounds)

### Common Patterns:

- Props tables always: Prop | Type | Description | Default (if applicable)
- Example intro: "`	sx" not "`jsx"
- Method examples use method chaining with "this"
- Animation props: durationInFrames (NOT duration)
- Coordinates: cx, cy (center), x, y, x1, y1, x2, y2 conventions
- Easing imports from '@elucim/core'
- Cross-references: "See [Guide Page Title](/path)" or "Next Steps" sections

## RECOMMENDATIONS FOR NEW EDITOR DOCUMENTATION

Based on analysis, new editor pages should:

1. **Match structure:** Add to appropriate category (likely new "Tools" section or "Advanced")
2. **Follow naming:** kebab-case filenames (e.g., editor-overview.md)
3. **Follow format:**
   - Start with intro paragraph
   - Use TSX code blocks for examples
   - Include prop/feature tables
   - Add Next Steps section with cross-links
   - Use 2-3 levels of headings
4. **Use conventions:**
   - Color palette: the 4 colors listed above
   - Link to guide pages for context
   - Include real-world examples
   - Cross-reference related components
5. **Update sidebar:** Add new section to config.ts like:
   {
     text: 'Editor & Tools',
     items: [
       { text: 'Editor Overview', link: '/guide/editor-overview' },
       { text: 'Live Preview', link: '/guide/live-preview' },
       // ... other editor docs
     ]
   }
